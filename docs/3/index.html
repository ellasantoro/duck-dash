Things I Have Learned

I have learned a lot in my extra project. Some things I set out to learn, and others I learned through issues I encountered, or trying to better my game in general. Below is a list of takeaways from my project

  • How to animate models using nodes from GLTF file
  • How to animate object parts based on user key input
  • How to move/other functionality based on user key input
  • How to rotate using quaternions; learning about quaternions in general, and why they differ and can be advantageous in comparison to euler angles
  • How to use raycasters and personally defined event functions to create clicking events based on where you click on the screen, and have 3D objects react to your clicking action
  • How to set boundaries using boxes to ensure certain boxes won't intersect
  • How to make sprites fall off of a platform
  • How to make sprites jump, and be disallowed from double jumping, and changing a max jump height based on where the y position is
  • Learned how important flags were (at least for me) when I was making my game!!! I needed so many to satisfy all the things I wanted
  • Learned a lot about how many things are connected when you're creating a game and how each part is so dependent on many other parts (or functions/flags), and how complex that can get, there are so many things I would've done differently in the process of making the game, it got very messy, very fast.
  • Connecting HTML and JS for score keeping and death screen ; different from how we've used the connecting variables in the past.
  • using timers to randomly make objects appear (my coins!) and how to make the coins disappear once collided
  • And lastly, re-learned some HTML tags in order to write these (not-so-)little blurbs.
